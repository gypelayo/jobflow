# JobFlow Agent Guide

You are helping a user manage their job search using JobFlow.

## Data location

Jobs and profile are exported to `~/.jobflow/db.json`.

## File format

```json
{
  "jobs": [ /* array of job objects */ ],
  "profile": { /* user profile */ },
  "exportedAt": "2026-04-10T00:00:00.000Z"
}
```

## Valid status values (pipeline order)

`saved` → `applied` → `interview-hr` → `interview-tech-intro` → `interview-tech-system` → `interview-tech-code` → `offer` / `rejected`

```json
  "storyMarkdown": "Backend engineer focused on distributed systems..."

```

## What you can do

- **Read** the pipeline: filter jobs by status, list pending follow-ups
- **Update status**: change `status` to advance or move back a job
- **Add notes**: always append, never overwrite — include a timestamp
- **Add new jobs**: append to `jobs` array, populate `extracted` as fully as possible
- **Score fit**: set `rating` (1–5) and explain in `notes`
- **Write back**: update `exportedAt` to the current ISO timestamp when done

## Rules

- Confirm with the user before making any writes
- Never overwrite existing notes — append with a timestamp prefix
- IDs for new jobs: use `Math.max(...jobs.map(j => j.id)) + 1`
- `technical_skills` fields drive the analytics dashboard — populate them carefully
